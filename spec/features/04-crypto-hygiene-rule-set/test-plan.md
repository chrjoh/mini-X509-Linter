# Test Plan: Crypto Hygiene Rule Set

## Scope

Verify the SPKI/signature accessors and the four hygiene lints (`no_sha1_signature`,
`rsa_key_min_2048`, `ecdsa_curve_allowlist`, `not_expired`), including correct
applicability for the wrong key type.

## Conventions

Per `.claude/rules/rust-testing-core.md`: SIFER, `.unwrap()`/`.unwrap_err()`, behaviour-
focused tests grouped per lint.

## Fixtures (`testdata/`)

- `hygiene_sha1_signature.pem`, `hygiene_rsa_1024.pem`, `hygiene_ecdsa_bad_curve.pem`.
- `expired.pem` (feature 01) covers `not_expired`; `good.pem` passes all hygiene lints.
- A P-256 EC cert for the ECDSA allowlist pass case (reuse `good.pem` if it is EC,
  otherwise add one). Regenerated by `testdata/generate.sh`.

## Integration Tests (`crates/linter/tests/hygiene.rs`)

- `no_sha1_signature`: flags SHA-1 fixture (message names the algorithm); passes `good.pem`.
- `rsa_key_min_2048`: flags RSA-1024 (message names bit length); `NotApplicable` on EC.
- `ecdsa_curve_allowlist`: flags non-allowlisted curve (message names curve);
  `NotApplicable` on RSA; passes a P-256 cert.
- `not_expired`: flags `expired.pem`; passes `good.pem` (consolidated-set sanity).

## Edge Cases

- Non-RSA / non-EC ("other") key algorithm → both key-strength lints `NotApplicable`,
  no panic.
- SHA-2 signature (e.g. sha256WithRSAEncryption) → `no_sha1_signature` passes.
- Curve emission limited by toolchain → documented hand-crafted fixture used, provenance
  noted in `generate.sh`.

## Verification Commands

```
cargo test
cargo clippy --all-targets -- -D warnings
cargo fmt --check
bash testdata/generate.sh
```

## Exit Criteria

All four hygiene lints validated; applicability correct for wrong key types; verification
commands pass.
